## ภาพรวม

ระบบทดสอบประสิทธิภาพ VGGT-MPS ประเมินการแลกเปลี่ยนระหว่างประสิทธิภาพและคุณภาพของกลไก Sparse Attention เทียบกับ Dense (Full) Attention

หลักการสำคัญ:

- **Dense Attention**: ความซับซ้อนหน่วยความจำ O(n²) - ทุกภาพเชื่อมต่อกับทุกภาพ
- **Sparse Attention**: ความซับซ้อนหน่วยความจำ O(n·k) - แต่ละภาพเชื่อมต่อกับ k ภาพที่ใกล้เคียงที่สุด

ผลลัพธ์คือสามารถประมวลผล **ภาพได้มากขึ้น 10-100 เท่า** บนฮาร์ดแวร์เดียวกัน

## โหมดการทดสอบ

### 1. `basic` - การทดสอบประสิทธิภาพพื้นฐาน

**คำสั่ง:**
```bash
vggt benchmark --mode basic --images 30 --compare dense,sparse
```

**สิ่งที่วัด:**
- เวลาประมวลผลของ dense เทียบกับ sparse
- การใช้หน่วยความจำ (บน CUDA)
- เฟรมต่อวินาที (FPS)

**ค่าวัดที่ได้:**
| ค่าวัด | คำอธิบาย |
|--------|----------|
| Time | เวลาประมวลผลทั้งหมด (วินาที) |
| FPS | จำนวนภาพที่ประมวลผลได้ต่อวินาที |
| Memory | หน่วยความจำ GPU ที่ใช้ (CUDA เท่านั้น) |
| Speedup | sparse เร็วกว่า dense กี่เท่า |

### 2. `consistency` - การทดสอบความสม่ำเสมอของคุณภาพ

**คำสั่ง:**
```bash
vggt benchmark --mode consistency \
    --images 5,10,20,30 \
    --compare dense,sparse \
    --metrics depth_l1,pose_rotation,pose_translation,chamfer
```

**สิ่งที่วัด:**
เปรียบเทียบคุณภาพผลลัพธ์ระหว่างวิธี dense และ sparse ในจำนวนภาพที่แตกต่างกัน

**ค่าวัดที่ได้:**
| ค่าวัด | คำอธิบาย | ต่ำ = ดี |
|--------|----------|----------|
| `depth_l1` | ค่าความผิดพลาดเฉลี่ยของการทำนายความลึก | ใช่ |
| `pose_rotation` | ค่าความผิดพลาดการหมุนของกล้อง | ใช่ |
| `pose_translation` | ค่าความผิดพลาดการเคลื่อนที่ของกล้อง | ใช่ |
| `chamfer` | ระยะ Chamfer ระหว่าง point cloud | ใช่ |

**หมายเหตุ:** ค่าจะแสดง "N/A" ในโหมดจำลอง เพราะ pose และ point cloud ต้องใช้โมเดล VGGT จริง

### 3. `ablation-k` - การทดสอบพารามิเตอร์ K-Nearest

**คำสั่ง:**
```bash
vggt benchmark --mode ablation-k \
    --images 30 \
    --sparse-k 3,5,10,15,20,30
```

**สิ่งที่วัด:**
พารามิเตอร์ k-nearest ส่งผลต่อคุณภาพและประสิทธิภาพอย่างไร

**ข้อมูลสำคัญ:**
- **k ต่ำ** = Sparsity สูง = เร็วขึ้นแต่อาจคุณภาพลดลง
- **k สูง** = Sparsity ต่ำ = ช้าลงแต่คุณภาพใกล้เคียง dense
- จุดสมดุลมักอยู่ที่ k=10-15

**ตารางผลลัพธ์:**
```
       k   Time(s)    Speedup   Sparsity   MemSavings     DepthL1
   dense      0.05      1.00x       0.0%         1.0x    0.000000  <- ค่าอ้างอิง
       5      0.03      1.67x      83.3%         6.0x    0.012345  <- Sparsity สูง
      10      0.04      1.25x      66.7%         3.0x    0.005678  <- สมดุล
      20      0.04      1.10x      33.3%         1.5x    0.001234  <- Sparsity ต่ำ
```

**วิธีตีความ:**
- `Sparsity 83.3%` หมายถึง 83.3% ของการเชื่อมต่อถูกตัดออก
- `MemSavings 6.0x` หมายถึงใช้หน่วยความจำน้อยกว่า dense 6 เท่า
- `DepthL1 0.012345` หมายถึงค่าความผิดพลาดความลึกเฉลี่ย 0.012345 เมตร

### 4. `ablation-tau` - การทดสอบพารามิเตอร์ Threshold

**คำสั่ง:**
```bash
vggt benchmark --mode ablation-tau \
    --images 30 \
    --threshold 0.3,0.5,0.7,0.8,0.9
```

**สิ่งที่วัด:**
ค่า covisibility threshold (τ) ส่งผลต่อการเชื่อมต่อระหว่างภาพอย่างไร

**ข้อมูลสำคัญ:**
- **τ ต่ำ (0.3)** = เก็บการเชื่อมต่อมากขึ้น = ใช้หน่วยความจำมากแต่คุณภาพดี
- **τ สูง (0.9)** = เก็บการเชื่อมต่อน้อยลง = ใช้หน่วยความจำน้อยแต่อาจพลาดความสัมพันธ์สำคัญ

**หลักการทำงานของ threshold:**
```
ถ้า similarity(ภาพ_i, ภาพ_j) >= τ:
    เก็บการเชื่อมต่อไว้ (attend กัน)
มิฉะนั้น:
    ตัดการเชื่อมต่อ (ไม่ attend กัน)
```

### 5. `ablation-mask` - การเปรียบเทียบประเภท Mask

**คำสั่ง:**
```bash
vggt benchmark --mode ablation-mask \
    --images 30 \
    --mask-types covisibility,random,sliding_window \
    --sparsity 0.56
```

**สิ่งที่วัด:**
เปรียบเทียบกลยุทธ์ต่างๆ ในการสร้าง sparse attention mask ที่ **ระดับ sparsity เดียวกัน**

**ประเภท Mask:**

| ประเภท | คำอธิบาย | เหมาะกับ |
|--------|----------|----------|
| `covisibility` | เชื่อมต่อภาพที่เห็นเนื้อหาฉากคล้ายกัน | ลำดับภาพจริง |
| `random` | เลือกการเชื่อมต่อแบบสุ่ม | เปรียบเทียบพื้นฐาน |
| `sliding_window` | เชื่อมต่อเฟรมที่อยู่ใกล้กันในลำดับ | ข้อมูลวิดีโอ/ลำดับเวลา |

**ทำไมสิ่งนี้สำคัญ:**
แม้ที่ sparsity เท่ากัน (เช่น 56%) **การเชื่อมต่อใด**ที่เราเก็บไว้ส่งผลต่อคุณภาพอย่างมาก Mask แบบ covisibility ดีกว่า random mask เพราะรักษาการเชื่อมต่อที่มีความหมายเชิงความหมาย

### 6. `compare-methods` - การเปรียบเทียบวิธีการทั้งหมด

**คำสั่ง:**
```bash
vggt benchmark --mode compare-methods \
    --images 30 \
    --methods dense,sliding_window,random,covisibility \
    --sparsity 0.5,0.6,0.7
```

**สิ่งที่วัด:**
การเปรียบเทียบครอบคลุมของทุกวิธีการในหลายระดับ sparsity

**ผลลัพธ์ประกอบด้วย:**
1. ตารางประสิทธิภาพสำหรับแต่ละคู่วิธี × sparsity
2. การจัดอันดับคุณภาพในแต่ละระดับ sparsity
3. ไฟล์ JSON สำหรับการวิเคราะห์เพิ่มเติม

**ตัวอย่างการจัดอันดับ:**
```
การจัดอันดับวิธีตามคุณภาพ (ค่าผิดพลาดต่ำสุดในแต่ละระดับ sparsity):
  50% sparsity: covisibility(0.0052) > sliding_window(0.0078) > random(0.0134)
  60% sparsity: covisibility(0.0061) > sliding_window(0.0089) > random(0.0156)
  70% sparsity: covisibility(0.0073) > sliding_window(0.0102) > random(0.0189)
```

**การตีความ:** Covisibility ให้ค่าผิดพลาดต่ำที่สุดอย่างสม่ำเสมอ ตามด้วย sliding_window และ random

## ความหมายของค่าวัด

### ค่าวัดหน่วยความจำ

| ค่าวัด | สูตร | คำอธิบาย |
|--------|------|----------|
| Dense Memory | O(n²) | เติบโตแบบกำลังสองตามจำนวนภาพ |
| Sparse Memory | O(n·k) | เติบโตแบบเส้นตรงตามจำนวนภาพ |
| Memory Savings | n/k | ใช้หน่วยความจำน้อยลงกี่เท่า |

**ตัวอย่าง:** ถ้า n=100 ภาพ และ k=10:
- Dense: 100 × 100 = 10,000 รายการ attention
- Sparse: 100 × 10 = 1,000 รายการ attention
- ประหยัด: 10 เท่า

### ค่าวัดคุณภาพ

| ค่าวัด | ช่วง | ความหมาย |
|--------|------|----------|
| Depth L1 | 0 ถึง ∞ | ค่าผิดพลาดความลึกเฉลี่ย (เมตร) 0 = สมบูรณ์แบบ |
| Sparsity | 0% ถึง 100% | เปอร์เซ็นต์การเชื่อมต่อที่ตัดออก สูง = sparse มาก |
| Connectivity | 0% ถึง 100% | การเชื่อมต่อเฉลี่ยต่อภาพ สูง = เชื่อมต่อมาก |

### ค่าวัดประสิทธิภาพ

| ค่าวัด | ความหมาย |
|--------|----------|
| Speedup | เร็วกว่า dense baseline กี่เท่า |
| Time | เวลาประมวลผลจริง (วินาที) |
| Theoretical | อ้างอิงจากการวิเคราะห์ความซับซ้อน |
| Measured | อ้างอิงจากการทดสอบจริง |

## คำแนะนำ

1. **สำหรับแอปพลิเคชัน real-time:** ใช้ k=5-10, ยอมรับ sparsity ~70%
2. **สำหรับการสร้าง 3D คุณภาพสูง:** ใช้ k=15-20, ยอมรับ sparsity ~50%
3. **สำหรับจำนวนภาพมากที่สุด:** ใช้ k=5, τ=0.8, ได้ sparsity ~85%
4. **ใช้ covisibility เสมอ** แทน random mask เมื่อเป็นไปได้

## สรุปสำคัญ

| หัวข้อ | Dense Attention | Sparse Attention |
|--------|-----------------|------------------|
| หน่วยความจำ | O(n²) - เติบโตเร็ว | O(n·k) - เติบโตช้า |
| รองรับภาพ | จำกัด (~50-100 ภาพ) | มาก (~500-1000 ภาพ) |
| คุณภาพ | ดีที่สุด | ใกล้เคียง (ถ้าใช้ covisibility) |
| ความเร็ว | พื้นฐาน | เร็วขึ้น 2-10 เท่า |

**ข้อสรุป:** Sparse Attention ด้วย covisibility mask เป็นทางเลือกที่ดีที่สุดสำหรับการประมวลผลภาพจำนวนมากบน Apple Silicon โดยรักษาคุณภาพใกล้เคียงกับ Dense Attention
